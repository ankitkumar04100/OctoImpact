# Contributing to OctoImpact

First of all, thank you for considering contributing to **OctoImpact**!  
Your contributions, suggestions, and bug reports are highly appreciated.

## How to Contribute

### 1. Fork the Repository
Click the **Fork** button at the top-right corner of the GitHub page to create your own copy.

### 2. Clone Your Fork
```bash
git clone https://github.com/yourusername/octoimpact.git
cd octoimpact
```

### 3. Create a Branch
Create a feature branch to work on your changes:
```bash
git checkout -b feature/your-feature-name
```

### 4. Make Changes

- Fix bugs, add features, or improve documentation.
- Follow code style (clean, readable, and modular).
- Ensure all existing functionality works before submitting.

### 5. Commit Your Changes
Use clear, concise commit messages:
```bash
git commit -m "Add feature: AI sustainability insights"
```

### 6. Push Changes
Push your branch to your fork:
```bash
git push origin feature/your-feature-name
```

### 7. Create a Pull Request

Go to the original repository and create a Pull Request.
- Provide a clear description of your changes.
- Mention any related issues (if applicable).

## Code of Conduct

Please read the Code of Conduct before contributing.
We maintain a respectful, inclusive, and collaborative environment.

## Reporting Issues

If you encounter a bug or want to request a feature:
1. Go to the Issues tab.
2. Click New Issue.
3. Provide a clear title and description with steps to reproduce.

## Guidelines

- Keep code modular and maintainable.
- Document new features.
- Test thoroughly before submitting.
- Respect the existing project architecture.
